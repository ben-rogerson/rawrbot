## Session: deploy-batch-three

Deployed 5 Hono+Lowdb SPAs to Cloudflare Workers (API) + Pages (frontend).

**Projects deployed:** webhook-inspector, grocery-list, airfryer-recipes, retro-shelf, woodwork-planner

**Pattern applied:**
- `src/app.ts` — extracted Hono<Env> app with cors; Env type exposes GOOGLE_GENERATIVE_AI_API_KEY binding
- `src/worker.ts` — CF Worker entry (`export default app`)
- `src/index.ts` — simplified to just `serve({ fetch: app.fetch, port })` with dotenv import
- `src/db.ts` — conditional singleton: `JSONFilePreset` (Node) vs `new Low(new Memory())` (CF Worker) via dynamic import and `process.versions?.node` check
- AI routes — `(c.env as Record<string, string|undefined> | undefined)?.GOOGLE_GENERATIVE_AI_API_KEY ?? process.env...`
- Frontend `api.ts` — `const BASE = import.meta.env.VITE_API_URL ?? '/api'` (grocery-list, airfryer-recipes, retro-shelf, woodwork-planner use /api proxy); webhook-inspector already used `?? ''` with direct-path proxies
- `wrangler.jsonc` — API Worker + Pages configs
- `env.d.ts` — added to 4 web packages; tsconfig `include` updated
- `.github/workflows/deploy.yml` — typecheck → parallel deploy-api + deploy-web

**Key decision:** grocery-list/airfryer-recipes/retro-shelf/woodwork-planner use Vite proxy at `/api` → strips prefix before forwarding. Fallback must be `/api` not `''` to preserve dev proxy routing.

**webhook-inspector specifics:** routes.ts had module-level `await db.read()` on singleton db. Moved to `getDb()` pattern and removed per-route `db.read()` calls (replaced with `const db = await getDb()` in each handler).

All 5 typechecks passed (0 errors). Portfolio updated with 5 new cards.
